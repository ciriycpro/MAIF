# MAIF Specification — Overview

**Версия:** 1.0
**Статус:** Опубликовано
**Дата:** 20 января 2026 года

## 1. Введение

Настоящий документ описывает требования к открытому стандарту индустриального мультиагентного слоя. Стандарт определяет шесть обязательных свойств реализаций и интерфейсные обязательства, обеспечивающие совместимость автономных агентов между системами разных вендоров и предприятий в индустриальной среде.

Стандарт применяется к системам, обрабатывающим автономные решения на объектах промышленной инфраструктуры, включая объекты критической информационной инфраструктуры (КИИ).

## 2. Терминология

**Агент** — программный компонент, способный к восприятию среды, принятию решений в рамках полномочий и выполнению действий, направленных на достижение поставленных целей.

**Автономный агент** — агент, принимающий решения без обязательного промежуточного подтверждения человеком, в пределах явно определённых полномочий.

**Прото-агент** — агент с рекомендательной функцией, не принимающий итогового решения без подтверждения уполномоченного оператора.

**Межагентный контур** — совокупность каналов и протоколов взаимодействия агентов разных систем, не входящих в общий внутренний периметр.

**Семантический протокол** — формализованное описание содержания сообщений между агентами, включающее типизацию, источник, полномочия и обязательства сторон.

**Реализация стандарта** — программный продукт, удовлетворяющий шести обязательным требованиям, определённым в разделах 01–06 настоящей спецификации.

## 3. Применимость

Стандарт применяется к проектированию и реализации:

- Индустриальных мультиагентных систем на объектах промышленной инфраструктуры.
- Систем кросс-вендорной и кросс-предприятной координации автономных решений.
- Архитектурных рамок совместимости для отраслевых программ цифровой трансформации.

Стандарт **не применяется** к:

- Системам без автономных компонентов (классическая АСУ ТП без агентного слоя).
- Системам с одиночным агентом без межсистемного взаимодействия.
- Корпоративным оркестраторам бизнес-процессов (BPM), не работающим с автономными решениями в реальном времени на физических активах.

## 5. Совместимость с существующими системами

Стандарт не требует переписывания существующих индустриальных систем. Интеграция с системами, не реализующими настоящий стандарт, выполняется через адаптеры. Адаптер обеспечивает преобразование интерфейсов существующей системы в семантический протокол стандарта без изменения внутренней реализации интегрируемой системы.
