# 05. Сквозной аудит

## Требование

Реализация стандарта **MUST** обеспечивать сквозной аудиторский след для каждого действия автономного агента. Аудиторский след **MUST** позволять восстановить полную траекторию принятия и выполнения решения.

## Минимальные обязательства реализации

Каждое действие агента **MUST** фиксироваться в аудиторском следе со следующими атрибутами:

1. **Trace ID** — идентификатор сквозной трассировки, общий для всех событий, связанных с одной логической операцией.
2. **Span ID** — идентификатор конкретного этапа обработки в пределах trace.
3. **Parent Span ID** — ссылка на родительский этап (при наличии).
4. **Идентификатор инициатора** — агент или внешняя система, инициировавшая операцию.
5. **Источник данных** — указание на данные, на основании которых принято решение.
6. **Применённый контракт** — ссылка на типизированный контракт, по которому обработано событие.
7. **Получатели** — агенты, получившие сообщение либо его результат.
8. **Подтверждения** — записи подтверждений уполномоченных операторов (при их применимости, см. раздел 04).
9. **Результат** — итог выполнения действия (успех / отказ / частичное выполнение).
10. **Метки времени** — для каждого события в пределах trace.

## Свойства аудиторского следа

Аудиторский след **MUST** обладать свойствами:

- **Целостность.** Записи защищены от изменения задним числом средствами криптографической верификации (хэш-цепочки, цифровые подписи или эквивалент).
- **Прослеживаемость.** Для любого записанного действия восстановимы все предшествующие шаги в пределах одной trace.
- **Хранение.** Срок хранения определяется отраслевым регламентом, но не менее срока, требуемого регуляторными нормами для соответствующей категории систем.

## Совместимость

Реализация **SHOULD** использовать OpenTelemetry или совместимые форматы трассировки для обеспечения интероперабельности с существующей наблюдаемостной инфраструктурой.
